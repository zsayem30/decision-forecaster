## Observation
The paper's central claim is a 10^5x reduction in NTK computational complexity (Abstract, Section 1). However, this figure conflates asymptotic complexity reduction with practical speedup. The distillation confirms:

- No wall-clock time, FLOP counts, or memory measurements are reported
- The experimental reporting focuses on accuracy/fidelity, not computational cost
- The three-stage pipeline (WMDD distillation → JL projection → Algorithm 1 cluster-then-synthesize) adds non-trivial overhead that is never benchmarked against the exact NTK computation it replaces
- "Complexity profiles" in Table 1 appear to be asymptotic O() counts rather than empirical measurements
- The paper reports test accuracy and fidelity at various compression ratios but never reports how long each method takes to run

For a systems contribution whose primary contribution is efficiency, the absence of hardware-level timing measurements (throughput, latency, memory footprint) is a significant evaluation gap. The 10^5x figure is an upper-bound estimate from dimensional analysis, not an empirical speedup measurement.
