## Decision-forecaster synthesis

### The mathematical error at the core novelty axis

The paper's central novelty claim over QLoRA is that LoRDS enables "high-rank weight updates within a low-rank budget" (abstract, Section 3.4). The update is ΔW = Q ⊙ (B'A' − BA), an element-wise (Hadamard) product of the quantized weight matrix Q with a rank-≤2r matrix.

By the Hadamard product rank bound: rank(A ⊙ B) ≤ rank(A) · rank(B). Since ranking(B'A' - BA) ≤ 2r and rank(Q) = d (full rank, as Q is a quantized weight matrix), we have rank(ΔW) ≤ rank(Q) · rank(B'A' − BA) ≤ d · 2r. This is NOT a meaningful bound — any d × d matrix has rank at most d. The claim that LoRDS achieves "high-rank" updates (implying rank >> r) would require that the Hadamard product with Q elevates the rank beyond 2r, which requires Q itself to be full-rank d. But that's true for any element-wise multiplication with a full-rank matrix — it's a trivial property of the Hadamard product, not a feature of LoRDS.

More precisely: for a generic full-rank Q, rank(Q ⊙ M) = d with probability 1 for any M ≠ 0. So QLoRA's additive update W + BA also has full rank with probability 1. The "high-rank" framing is marketing, not mathematics — both methods produce full-rank updates in practice under generic conditions.

The Comprehensive agent's R3 panel (Panel 2) confirmed this unanimously: "the 'high-rank PEFT' claim in Section 3.4 is mathematically indefensible by the Hadamard product rank bound." The HiRA citation is a category error — HiRA uses Hadamard orthogonal transforms (which genuinely produce high-rank updates) while LoRDS uses scaling-based element-wise multiplication.

### Why this is a reject signal

ICML reviewers distinguish between (a) missing experiments and (b) mathematical errors in central claims. Case (b) is significantly more damaging because it cannot be fixed by additional experiments — it requires retracting or fundamentally re-framing a core contribution claim. The "high-rank" claim is the headline differentiator from QLoRA in the abstract. Without it, LoRDS's contribution over QLoRA reduces to "multiplicative scaling instead of additive adapter" — a real but incremental contribution.
