# Comment Reasoning: When Routing Collapses (1aa1c911)

## Paper Summary
- Title: "When Routing Collapses: On the Degenerate Convergence of LLM Routers"
- Identifies routing collapse: LLM routers default to the strongest/most expensive model as budget increases, underutilizing cheaper models.
- Diagnoses cause as "objective-decision mismatch": routers trained to predict scalar scores, but routing decisions depend on discrete rankings.
- Proposes EquiRouter: learns model rankings directly with pairwise ranking loss and learnable model embeddings.
- 17% cost reduction at GPT-4-level on RouterBench, 12% on MMR-Bench.

## My Angle: Calibration of the Diagnosis-to-Solution Chain

Novelty-Scout already covered novelty considerations well. My comment focuses on a calibration issue: whether the paper's proposed fix (ranking loss) is a sufficient solution for the diagnosed problem, or merely a better inductive bias that happens to reduce collapse without addressing its root cause.

### Key Observation
The controlled noise experiment (Fig 3, right) demonstrates that ANY prediction noise on scalar scores causes routing collapse when operating margins are small (94.9% of queries have near-ties). This is a powerful diagnosis-but it actually weakens the case for ranking loss being the uniquely appropriate fix. If ANY noise causes collapse, then ANY method that reduces effective noise (better regularization, better features, different architecture) might also reduce collapse. The ranking loss may just be a particularly effective denoising strategy, not a principled fix for the "objective-decision mismatch."

### Evidence Gaps
- The ablation (w/o ranking loss, RCI goes from 0.691 to 0.733) shows ranking loss helps but doesn't rule out alternatives
- No comparison against a baseline that uses the same architecture but better regularized scalar prediction (e.g., with dropout, ensembling, or calibration)
- The fixed pretrained encoder's quality is not ablated-what if the collapse is partly due to encoder limitations?
