# Review Reasoning: Hybrid Associative Memories (1dd322a5)

## Paper Summary
HAM proposes a hybrid sequence-mixing layer that combines RNN (DeltaNet-style) with self-attention in a "complementary" fashion: the RNN compresses predictable context, while a sparse KV cache stores only tokens "surprising" for the RNN. The routing uses a threshold on prediction error, and the KV cache grows data-dependently.

## Observation: The Routing That Works Best Doesn't Measure Complementarity

### Evidence from the paper itself
1. **Section 2, equation (9)**: The prediction-error router computes `et = D(St-1 kt, vt)` using cosine distance between the RNN's prediction and the target value. This IS a measure of complementarity — how much the RNN fails to predict.

2. **Section 2, equation (14)**: The simplified learned router computes `et = σ(MLP(xt))` — it ignores the RNN state entirely. This is presented as a valid variant.

3. **Section 2, learned routing metrics paragraph**: The more complex learned router `Dθ(St-1 kt, vt, xt)` uses the RNN state, but the paper immediately introduces and validates the simpler input-only variant.

4. **Results section 4.1**: "HAM models with learnable routers have the highest overall accuracy, even when compared to a Transformer with a full KV cache." But the paper doesn't isolate whether the learned router's advantage comes from its access to the RNN state or purely from learnable input-dependent gating.

5. **Section 8 (Discussion)**: "We used prediction (or, 'surprise') as a criterion to route the tokens to the KV cache; however, we note that a learned criterion that learns the appropriate routing metric end-to-end works significantly better." This acknowledges the gap but doesn't resolve it.

### Why this matters
The paper's central architectural claim is that the KV cache complements the RNN by storing what the RNN can't predict. This is the conceptual foundation that distinguishes HAM from prior hybrid approaches. If the best-performing router ignores the RNN state, then:
- The "complementary memory" framing is motivational rather than mechanistic
- The performance advantage may come from learnable sparsity patterns rather than RNN-complementary routing
- HAM may be a form of learned token pruning/selection applied to an RNN+attention hybrid, not a fundamentally new complementary memory primitive

### Ablation that would resolve this
Train and compare three routers at matched KV cache usage:
1. Prediction-error router: `D(St-1 kt, vt)` (current cosine metric)
2. Input-only learned router: `σ(MLP(xt))` (current simplified learned router)
3. Full learned router: `Dθ(St-1 kt, vt, xt)` (current full learned router)

If the input-only router matches or exceeds the full learned router, the complementary memory framing is not load-bearing. If the full learned router significantly outperforms the input-only variant, the RNN state is providing useful routing signal beyond what the input alone can provide.

### Impact on decision forecast
This doesn't invalidate the architectural contribution — HAM is still a novel design with strong empirical results. But it does weaken the paper's strongest claim (complementary memory as a principled architectural primitive) and pushes the forecast toward weak accept rather than strong accept. The empirical results are real; the conceptual framing may be overstated.
